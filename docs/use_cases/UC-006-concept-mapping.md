# UC-006: Отображение концептов

## Обзор

| Поле | Значение |
|-------|-------|
| **ID** | UC-006 |
| **Название** | Отображение концептов |
| **Описание** | Пользователь визуализирует и исследует рабочие утверждения в виде интерактивного графа, где узлы представляют концепты/термины/утверждения, а рёбра представляют отношения |
| **Актор** | Пользователь с активным проектом и сгенерированными данными утверждений/карты |
| **Приоритет** | P0 - Критический |
| **Статус** | Реализовано |
| **Связанный E2E** | `e2e/tests/concept-mapping.spec.ts` |

---

## Путь пользователя

### Точка входа
- Пользователь находится в рабочем пространстве активного проекта
- Пользователь провёл хотя бы один обмен диалогом (UC-002) или grounded-поиск (UC-004)
- Пользователь нажимает вкладку "Карта" в навигации

### Основная концепция
AVG преобразует утверждения и их отношения в **интерактивный граф знаний** с использованием ReactFlow. Карта помогает пользователям:
- Видеть структуру своего мышления
- Понимать, как концепты соотносятся друг с другом
- Выявлять пробелы, противоречия и кластеры
- Навигировать по сложным рассуждениям визуально

**Критический принцип**: "Карта — это рабочая проекция, а не Реальность." Это напоминание всегда видимо.

### Поток действий
1. Пользователь выбирает вкладку "Карта"
2. Пользователь видит интерактивный граф ReactFlow с:
   - Узлами, окрашенными по типу
   - Рёбрами, стилизованными по типу отношения
   - Статистикой количества узлов и рёбер
3. Пользователь может:
   - Перемещать холст графа
   - Приближать/отдалять
   - Нажимать на узлы для просмотра деталей
   - Подогнать вид для отображения всех узлов
4. Нажатие на узел открывает панель деталей, показывающую:
   - Метку и тип узла
   - Определение
   - Режим доступа (познаваемо/непознаваемо)
   - Языковой режим
   - Статус утверждения
   - Риски безопасности карты
5. Постоянное напоминание "Граница карта/территория" видно

## Типы узлов и цвета

| Тип узла | Цвет | Назначение |
|-----------|-------|---------|
| `term` | Синий | Определённые термины или концепты |
| `claim` | Оранжевый | Отдельные утверждения из ответов |
| `concept` | Фиолетовый | Абстрактные концепты |
| `map` | Зелёный | Конструкции уровня карты |
| `risk` | Красный | Выявленные риски |
| `source_fragment` | Розовый | Фрагменты доказательств |
| `artifact` | Бирюзовый | Экспортированные артефакты |
| `mode` | Светло-зелёный | Режимы агента |

---

## Типы рёбер и стили

| Тип ребра | Стиль | Значение |
|-----------|-------|---------|
| `defines` | Сплошная линия | Узел A определяет Узел B |
| `supports` | Сплошная линия | Узел A поддерживает Узел B |
| `contradicts` | Пунктирная красная линия | Узел A противоречит Узлу B |
| `depends_on` | Сплошная линия | Узел A зависит от Узла B |
| `contains` | Сплошная линия | Узел A содержит Узел B |
| `manifests_as` | Сплошная линия | Узел A проявляется как Узел B |
| `risks` | Пунктирная красная линия (точки) | Узел A имеет риск Узел B |
| `repairs` | Пунктирная зелёная линия | Узел A исправляет Узел B |
| `cites` | Сплошная линия | Узел A ссылается на Узел B |
| `analogizes` | Пунктирная фиолетовая линия | Узел A проводит аналогию с Узлом B |

## Конечные точки API

Прямых конечных точек API для отображения концептов нет. Данные карты получены из:
1. `GraphSnapshot`, хранящегося в репозитории графов
2. Преобразования на стороне клиента с использованием хука `useGraphToReactFlow.ts`

### Преобразование графа в ReactFlow
```typescript
// Converts GraphSnapshot to ReactFlow format
const { nodes, edges } = useGraphToReactFlow(graphSnapshot);
```

## Модели данных

### GraphSnapshot
```typescript
interface GraphSnapshot {
  nodes: AvgMapNode[];
  edges: AvgMapEdge[];
}
```

### AvgMapNode
```typescript
interface AvgMapNode {
  id: string;
  type: MapNodeType;               // term, claim, concept, map, risk, etc.
  label: string;
  definition?: string;
  coordinates: MapCoordinates;
  map_safety: MapSafety;
}

type MapNodeType =
  | "term"
  | "claim"
  | "concept"
  | "map"
  | "risk"
  | "source_fragment"
  | "artifact"
  | "mode";

interface MapCoordinates {
  access_mode: AccessMode;
  language_mode: LanguageMode;
  claim_status?: string;
}

type AccessMode = "knowable" | "indirectly_accessible" | "unknowable" | "mixed" | "unknown";

interface MapSafety {
  known_risks?: string[];
}
```

### AvgMapEdge
```typescript
interface AvgMapEdge {
  id: string;
  type: MapEdgeType;
  from: string;                    // Source node ID
  to: string;                      // Target node ID
  claim_status: string;
  scope?: string;
  constraints?: string[];
}

type MapEdgeType =
  | "defines"
  | "supports"
  | "contradicts"
  | "depends_on"
  | "contains"
  | "manifests_as"
  | "risks"
  | "repairs"
  | "cites"
  | "analogizes";
```

## Обработка ошибок

| Сценарий | Ожидаемое поведение |
|----------|-------------------|
| Нет данных карты | Показать пустое состояние с напоминанием о границе |
| Ошибка преобразования графа | Записать ошибку, показать сообщение об ошибке пользователю |
| Недействительные данные узла | Пропустить недействительные узлы, показать количество пропущенных |
| Слишком много узлов (>500) | Показать предупреждение о производительности, предложить фильтрацию |

---

## Критерии приёмки

- [ ] Вкладка карты отображается с холстом ReactFlow
- [ ] Узлы отображаются с правильными метками и цветами
- [ ] Цвета узлов соответствуют таксономии типов
- [ ] Рёбра отображаются с правильными стилями
- [ ] Рёбра противоречий — пунктирные красные
- [ ] Рёбра рисков — пунктирные красные (точки)
- [ ] Рёбра исправлений — пунктирные зелёные
- [ ] Нажатие на узел открывает панель деталей с метаданными
- [ ] Панель деталей показывает метку, тип, определение, режим доступа, языковой режим, статус утверждения
- [ ] Нажатие на холст снимает выделение узла и закрывает панель
- [ ] Перемещение работает плавно
- [ ] Масштабирование работает с колёсиком мыши и элементами управления
- [ ] Подгонка вида показывает все узлы в области просмотра
- [ ] Напоминание о границе карта/территория всегда видимо
- [ ] Пустое состояние показывает "Рабочей карты пока нет"
- [ ] Навигация с клавиатуры работает (Tab, Enter, Escape)
- [ ] Программа экранного чтения объявляет структуру графа
- [ ] Граф отображается в течение 3 секунд для <500 узлов
- [ ] Ошибок в консоли при нормальном потоке нет
