# UC-001: Создание проекта

## Обзор

|Поле|Значение|
|-|-|
|**ID**|UC-001|
|**Название**|Создание проекта|
|**Описание**|Пользователь создаёт новый проект AVG для начала сессий структурированного мышления|
|**Актор**|Любой пользователь (первый визит или повторный)|
|**Приоритет**|P0 — Критический|
|**Статус**|Реализовано|
|**Связанные E2E**|`e2e/tests/project-creation.spec.ts`|

\---

## Путь пользователя

### Точка входа

* Пользователь открывает веб-приложение AVG Lab по адресу `http://localhost:3000`
* Пользователь видит **лендинг** со слоганом "Think Better. Think Structured."

### Секции лендинга

1. **Hero-секция** — Заголовок + кнопка CTA "Start Thinking"
2. **Основное обещание** — Философия "Карта — это не территория"
3. **Сетка возможностей** (6 карточек):

   * Структурированное мышление
   * Креативные направления
   * Карта концептов
   * Валидация утверждений
   * Обоснованные доказательства
   * Экспортируемые артефакты
4. **Таблица отличий** — Сравнение AVG vs Чат-бот vs Поиск
5. **Карточки персон** (4 типа):

   * Основатели и креаторы
   * Писатели и мыслители
   * Исследователи
   * Фасилитаторы команд
6. **Footer CTA** — "Ready to Think With Discipline?" + кнопка "Create Project"

### Поток действий

1. Пользователь нажимает кнопку "Start Thinking" или "Create Project"
2. Система генерирует уникальный **Project ID** (на основе временной метки)
3. Система генерирует уникальный **Session ID**
4. Система переходит от LandingPage к WorkspaceShell
5. Пользователь входит в рабочее пространство с 6 навигационными вкладками

\---

## Тест-сценарии

### TC-001.1: Отображение лендинга

**Предусловие**: Отсутствует (чистый браузер, нет проекта)

**Шаги**:

1. Перейти по адресу `http://localhost:3000`
2. Убедиться, что hero-секция с "Think Better. Think Structured." видна
3. Убедиться, что все 6 карточек возможностей отображаются
4. Убедиться, что все 4 карточки персон отображаются
5. Убедиться, что кнопка CTA "Start Thinking" кликабельна
6. Убедиться, что кнопка "Create Project" кликабельна

**Ожидаемый результат**:

* Лендинг рендерится без ошибок
* Все секции видны и правильно стилизованы
* Обе кнопки CTA активны

**Приоритет**: P0

\---

### TC-001.2: Создание проекта через "Start Thinking"

**Предусловие**: Пользователь находится на лендинге

**Шаги**:

1. Нажать кнопку "Start Thinking" в hero-секции
2. Наблюдать переход к WorkspaceShell
3. Убедиться, что в заголовке отображается новый Project ID
4. Убедиться, что в заголовке отображается новый Session ID
5. Убедиться, что бейдж "Local only" виден
6. Убедиться, что присутствуют 6 навигационных вкладок: Dialogue, Documents, Retrieval, Claim Review, Map, Artifacts

**Ожидаемый результат**:

* Project ID сгенерирован (не пустой, уникальный формат)
* Session ID сгенерирован (не пустой, уникальный формат)
* WorkspaceShell рендерится со всеми 6 вкладками
* Заголовок показывает метаданные проекта

**Приоритет**: P0

### TC-001.3: Создание проекта через "Create Project"

**Предусловие**: Пользователь находится на лендинге

**Шаги**:

1. Прокрутить до секции CTA в footer
2. Нажать кнопку "Create Project"
3. Наблюдать переход к WorkspaceShell
4. Убедиться, что Project ID и Session ID отображаются в заголовке
5. Убедиться, что вкладки рабочего пространства доступны

**Ожидаемый результат**:

* Аналогично TC-001.2
* Обе кнопки приводят к одинаковому опыту работы

**Приоритет**: P0

\---

### TC-001.4: Генерация уникальных ID

**Предусловие**: Пользователь находится на лендинге

**Шаги**:

1. Нажать "Start Thinking" для создания проекта A
2. Записать Project ID и Session ID
3. Обновить браузер (или вернуться на лендинг)
4. Нажать "Create Project" для создания проекта B
5. Записать Project ID и Session ID
6. Сравнить ID проекта A и проекта B

**Ожидаемый результат**:

* Project ID A != Project ID B
* Session ID A != Session ID B
* ID уникальны для каждого события создания

**Приоритет**: P0

\---

### TC-001.5: Технические детали рабочего пространства

**Предусловие**: Пользователь находится в активном рабочем пространстве

**Шаги**:

1. Найти секцию технических деталей в заголовке
2. Нажать для раскрытия технических деталей
3. Убедиться, что Project ID отображается
4. Убедиться, что Session ID отображается
5. Убедиться, что версия контракта показана как "mvp-5"

**Ожидаемый результат**:

* Раскрытие/сворачивание технических деталей работает
* Все поля метаданных отображаются корректно
* Версия контракта показывает "mvp-5"

**Приоритет**: P1

### TC-001.6: Лендинг после создания проекта

## API эндпоинты

Во время создания проекта API-вызовы не выполняются. Project ID и Session ID генерируются на стороне клиента.

\---

## Модели данных

### Генерируемые данные

```typescript
interface ProjectRecord {
  id: string;      // Уникальный ID на основе временной метки
  name: string;    // Отображаемое имя проекта
}

interface SessionRecord {
  id: string;      // Уникальный ID на основе временной метки
  projectId: string;
  title: string;   // Отображаемое название сессии
}
```

\---

## Обработка ошибок

|Сценарий|Ожидаемое поведение|
|-|-|
|Обновление браузера|Состояние проекта/сессии должно сохраниться или корректно сброситься|
|Многократные быстрые клики|Должен создаваться только один проект на событие клика|
|Сеть недоступна|Создание проекта локальное, должно работать|

\---

## Критерии приёмки

* \[ ] Лендинг рендерится со всеми видимыми секциями
* \[ ] Обе кнопки CTA создают новый проект
* \[ ] Project ID и Session ID уникальны при создании
* \[ ] WorkspaceShell рендерится с 6 навигационными вкладками
* \[ ] Бейдж "Local only" отображается в заголовке
* \[ ] Технические детали раскрываются/сворачиваются
* \[ ] Версия контракта отображается как "mvp-5"
* \[ ] Нет ошибок в консоли при создании проекта
* \[ ] Время загрузки страницы < 3 секунд
