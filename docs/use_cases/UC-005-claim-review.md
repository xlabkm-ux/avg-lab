# UC-005: Проверка утверждений

## Обзор

| Поле | Значение |
|-------|-------|
| **ID** | UC-005 |
| **Название** | Проверка утверждений |
| **Описание** | Пользователь просматривает все утверждения, созданные во время диалоговых сессий, каждое из которых аннотировано статусом утверждения, языковым режимом, рисками валидации и предложениями по исправлению |
| **Актор** | Пользователь с активным проектом и сгенерированными утверждениями |
| **Приоритет** | P0 - Критический |
| **Статус** | Реализовано |
| **Связанный E2E** | `e2e/tests/claim-review.spec.ts` |

---

## Путь пользователя

### Точка входа
- Пользователь находится в активном рабочем пространстве проекта
- У пользователя был как минимум один диалоговый обмен (UC-002) или обоснованный поиск (UC-004)
- Пользователь нажимает вкладку "Проверка утверждений" в навигации

### Основная концепция
AVG извлекает отдельные **утверждения** из каждого ответа и подвергает каждое тщательной эпистемической проверке. Пользователи могут просматривать:
- **Что** утверждается
- **Насколько уверена** система в этом
- **Какие риски** связаны с верой в это или действиями на его основе
- **Как исправить** проблемные утверждения

Это основной механизм предотвращения галлюцинаций, избыточных утверждений и дрейфа метафоры в факт.

### Поток действий
1. Пользователь выбирает вкладку "Проверка утверждений"
2. Пользователь видит список всех утверждений, извлечённых из ответов сессии
3. Каждое утверждение показывает:
   - Текст утверждения
   - Значок статуса утверждения (цветовой код)
   - Значок языкового режима
   - Индикаторы риска (если есть)
4. Пользователь нажимает для раскрытия утверждения
5. Раскрытый вид показывает:
   - Область применения (граница/контекст)
   - Список выявленных рисков
   - Предложение по исправлению (если есть риски)
   - Ссылки на источники (если обосновано)
6. Общий статус показывает "валидировано" или "требуется_исправление"

## Таксономия статусов утверждений

| Статус | Значение | Визуализация | Требуемое действие |
|--------|---------|--------|-----------------|
| `definition` | Установленное определение | Синий/нейтральный | Нет |
| `working_distinction` | Рабочее различие | Зелёный/нейтральный | Нет |
| `operational_marker` | Операциональный маркер | Бирюзовый/нейтральный | Нет |
| `indirect_sign` | Косвенный признак | Фиолетовый/нейтральный | Осведомлённость |
| `hypothesis` | Гипотеза (неуверенность) | Жёлтый/предупреждение | scrutiny (внимательная проверка) |
| `metaphor_only` | Только метафора (не факт) | Оранжевый/предупреждение | **Необходимо не воспринимать как факт** |
| `prohibited_positive_claim` | Запрещено | Красный/критический | **Необходимо отклонить** |
| `boundary_statement` | Пограничное утверждение | Серый/информация | Признать ограничения |

---

## Таксономия языковых режимов

| Режим | Значение |
|------|---------|
| `direct_description` | Прямое фактическое утверждение |
| `operational_description` | Описывает, как что-то работает |
| `conditional_description` | Описывает при определённых условиях |
| `metaphor` | Использует метафорический язык |
| `symbolic_pointer` | Указывает на что-то символическое |
| `silence_required` | Не может быть выражено языком |

---

## Уровни рисков валидации

| Уровень | Цвет | Значение |
|-------|-------|---------|
| `low` | Зелёный | Минимальная проблема |
| `medium` | Жёлтый | Некоторая неуверенность |
| `high` | Оранжевый | Существенный риск |
| `critical` | Красный | Не следует доверять |

---

## Типы рисков безопасности карты

| Риск | Описание |
|------|-------------|
| `reductionism` | Упрощение сложных явлений |
| `map_territory_substitution` | Путаница модели с реальностью |
| `dogma` | Представление гипотезы как факта |
| `overgeneralization` | Широкие утверждения на основе ограниченных доказательств |
| `category_error` | Смешение понятий из несовместимых областей |
| `false_precision` | Подразумеваемая большая уверенность, чем существует |

## Сценарии тестирования

### TC-005.1: Отображение вкладки "Проверка утверждений"
**Предусловие**: Пользователь находится в активном рабочем пространстве проекта со сгенерированными утверждениями

**Шаги**:
1. Нажмите вкладку "Проверка утверждений" в навигации
2. Убедитесь, что ClaimReviewPanel отображается
3. Убедитесь, что заголовок счётчика утверждений отображается: "N утверждений"
4. Убедитесь, что индикатор статуса показывает data-status="validated" или "needs_repair"
5. Убедитесь, что список утверждений отображается

**Ожидаемые результаты**:
- Вкладка "Проверка утверждений" выбрана и активна
- Счётчик утверждений точен
- Индикатор статуса отражает общее состояние утверждений
- Утверждения перечислены по порядку
- Нет ошибок в консоли

**Приоритет**: P0

### TC-005.2: Отображение карточки утверждения (свёрнуто)
**Предусловие**: Утверждения присутствуют

**Шаги**:
1. Найдите каждую карточку утверждения в списке
2. Убедитесь, что каждая свёрнутая карточка отображает:
   - Текст утверждения (усечённый, если длинный)
   - Значок статуса утверждения с правильным статусом и цветом
   - Значок языкового режима с правильным режимом
   - Индикатор риска (если утверждение имеет риски, показать значок/значок предупреждения)
3. Убедитесь, что карточки кликабельны для раскрытия

**Ожидаемые результаты**:
- Все утверждения отображаются
- Значки показывают правильные значения с правильным стилем
- Утверждения с рисками визуально отличаются
- Усечение карточек работает для длинных утверждений

**Приоритет**: P0

| `metaphor_only` | Оранжевый |
| `prohibited_positive_claim` | Красный |
| `boundary_statement` | Серый |

**Ожидаемые результаты**:
- Каждый значок использует правильный цвет
- Цвета различимы друг от друга
- Контраст цветов соответствует стандартам WCAG AA

**Приоритет**: P0

### TC-005.4: Раскрытие карточки утверждения
**Предусловие**: Утверждения присутствуют

**Шаги**:

### TC-005.5: Подсветка рисков
**Предусловие**: Существуют утверждения с рисками

**Шаги**:
1. Найдите утверждения со значками или индикаторами рисков
2. Убедитесь, что рискованные утверждения имеют дополнительную визуальную обработку:
   - Подсветка фона или граница
   - Значок предупреждения
   - Значок риска с меткой типа риска
3. Раскройте рискованное утверждение
4. Убедитесь, что список рисков показывает каждый тип риска:
   - `reductionism`
   - `map_territory_substitution`
   - `dogma`
   - и т.д.

**Ожидаемые результаты**:
- Рискованные утверждения визуально заметны
- Типы рисков — удобочитаемые метки
- Несколько рисков отображаются в виде списка
- Утверждения со статусом `metaphor_only` или `prohibited_positive_claim` особенно выделены

**Приоритет**: P0

**Ожидаемые результаты**:
- Предложение по исправлению чётко обозначено

### TC-005.8: Индикатор общего статуса

**Ожидаемые результаты**:
- Общий статус точно отражает состояние утверждений

## Конечные точки API

Прямые конечные точки API для проверки утверждений отсутствуют. Утверждения извлекаются на стороне клиента из `AvgStructuredResponse` с помощью хука извлечения утверждений (`useClaimExtraction.ts`).

### Поток извлечения утверждений
1. Ответ получен от API диалога/поиска
2. `extractClaimsFromAvgResponse()` обрабатывает ответ
3. Хук `useClaimExtraction.ts` мемоизирует извлечение
4. `ClaimExtractionReport` передаётся в ClaimReviewPanel

interface AvgClaim {

## Обработка ошибок

| Сценарий | Ожидаемое поведение |
|----------|-------------------|
| Утверждения не сгенерированы | Показать дружелюбное пустое состояние |
| Ошибка извлечения утверждений | Записать ошибку, показать предупреждение пользователю |
| Неверные данные утверждения | Пропустить неверные утверждения, показать количество пропущенных |
| Ошибка сети во время получения источников | Показать частичные данные, отметить отсутствующие источники |

## Критерии приёмки

- [ ] Вкладка "Проверка утверждений" отображается со списком утверждений
- [ ] Заголовок счётчика утверждений точен
- [ ] Индикатор общего статуса показывает validated/needs_repair
- [ ] Каждая карточка утверждения отображает текст, значок статуса и языковой режим
- [ ] Значки статуса утверждения используют правильную цветовую кодировку
- [ ] Утверждения с рисками визуально выделены
- [ ] Нажатие на карточку утверждения раскрывает/сворачивает детали
- [ ] Раскрытый вид показывает область применения, риски, исправление и ссылки на источники
- [ ] Предложения по исправлению чётко обозначены и содержат указания к действию
- [ ] Запрещённые утверждения имеют крайнюю визуальную обработку
- [ ] Утверждения "только метафора" отличаются от фактов
- [ ] Пустое состояние показывает "Нет утверждений для проверки"
- [ ] Навигация с клавиатуры работает (Tab, Enter, Escape)
- [ ] Экранное читательское устройство объявляет метаданные утверждения
- [ ] Цвет не является единственным индикатором статуса/риска
- [ ] Нет ошибок в консоли при нормальном потоке
