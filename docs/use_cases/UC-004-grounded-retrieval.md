# UC-004: Укоренённый поиск

## Обзор

| Поле | Значение |
|-------|-------|
| **ID** | UC-004 |
| **Название** | Укоренённый поиск |
| **Описание** | Пользователь задаёт вопросы по зарегистрированным документам проекта; AVG сначала ищет доказательства, показывает цитаты и формирует ответы, основанные на доказательствах, с уровнями уверенности и сигналами риска |
| **Участник** | Пользователь с активным проектом и зарегистрированными документами |
| **Приоритет** | P0 — Критический |
| **Статус** | Реализовано |
| **Связанный E2E** | `e2e/tests/grounded-retrieval.spec.ts` |

---

## Пользовательский сценарий

### Точка входа
- Пользователь находится в рабочем пространстве активного проекта
- Пользователь зарегистрировал хотя бы один документ (UC-003)
- Пользователь нажимает вкладку «Поиск» в навигации

### Основная концепция
В отличие от стандартных чат-ботов на базе ИИ, которые генерируют ответы на основе обучающих данных, укоренённый поиск AVG **сначала ищет предоставленные пользователем доказательства**. Система:
1. Ищет в зарегистрированных документах релевантные доказательства
2. Показывает панель цитирования с найденными фрагментами и оценками уверенности
3. Генерирует ответ, основанный на найденных доказательствах
4. Отображает ответ со статусом утверждения, языковым режимом, риском валидации и маркерами риска

### Поток действий
1. Пользователь выбирает вкладку «Поиск»
2. Пользователь видит интерфейс GroundedRetrievalFlow:
   - Текстовое поле для запроса
   - Кнопка отправки «Спросить с доказательствами»
3. Пользователь вводит вопрос по зарегистрированным документам
4. Пользователь отправляет запрос
5. Система ищет в документах и отображает:
   - **Бейдж уверенности поиска** (отсутствует/низкая/средняя/высокая)
   - **Пограничное утверждение**, объясняющее ограничения доказательств
6. Появляется **панель цитирования** с фрагментами найденных доказательств
7. Появляется **панель укоренённого ответа** с ответом, основанным на доказательствах
8. Если доказательства не найдены, система показывает состояние «Отсутствующие доказательства» с предложениями

## Тестовые сценарии

### TC-004.1: Отображение вкладки «Поиск»
**Предусловие**: Пользователь находится в рабочем пространстве активного проекта

**Шаги**:
1. Нажать вкладку «Поиск» в навигации
2. Убедиться, что интерфейс GroundedRetrievalFlow отображается
3. Убедиться, что присутствуют следующие элементы:
   - Текстовое поле для запроса с заполнителем
   - Кнопка отправки «Спросить с доказательствами»
   - Пустая область для результатов

**Ожидаемые результаты**:
- Вкладка «Поиск» выбрана и активна
- Интерфейс запроса отображается корректно
- Кнопка отправки включена
- Ошибок в консоли нет

**Приоритет**: P0

**Шаги**:

**Ожидаемые результаты**:
- Кнопка отправки показывает состояние загрузки
- Система ищет в документах
- Результаты отображаются с уверенностью поиска и цитатами
- Укоренённый ответ появляется с метаданными

**Приоритет**: P0

### TC-004.4: Отображение уверенности поиска
**Предусловие**: Запрос был отправлен и есть результаты

**Шаги**:
1. Найти бейдж уверенности поиска
2. Проверить уровень уверенности и цветовую кодировку:

| Уверенность | Цвет | Значение |
|------------|-------|---------|
| `none` | Серый | Доказательства не найдены |
| `low` | Красный | Слабое совпадение, низкая релевантность |
| `medium` | Жёлтый | Умеренное совпадение, частичные доказательства |
| `high` | Зелёный | Сильное совпадение, надёжные доказательства |

3. Убедиться, что пограничное утверждение отображается под бейджем уверенности

**Ожидаемые результаты**:
- Бейдж уверенности отображает корректный уровень
- Цветовая кодировка соответствует уровню уверенности
- Пограничное утверждение объясняет, что означает уверенность

**Приоритет**: P0

---

### TC-004.5: Отображение панели цитирования
**Предусловие**: Запрос вернул результаты поиска

**Шаги**:
1. Убедиться, что панель цитирования появляется под запросом
2. Убедиться, что отображается количество результатов: «N результатов»
3. Убедиться, что присутствует выпадающий список фильтра уверенности (Все/Высокая/Средняя/Низкая) с количеством
4. Убедиться, что каждая цитата отображает:
   - Метку источника
   - Бейдж уверенности
   - Оценку релевантности
   - ID фрагмента, ID цитаты, ID документа
   - Совпавший текст (обрезан до 150 символов)
5. Убедиться, что цитаты можно развернуть/свернуть

**Ожидаемые результаты**:
- Панель цитирования отображается со всеми элементами
- Количество результатов точное
- Выпадающий список фильтра показывает количество по уровням
- Цитаты отображают полные метаданные

**Приоритет**: P0

### TC-004.6: Отображение панели укоренённого ответа
**Предусловие**: Запрос вернул результаты поиска

**Шаги**:
1. Убедиться, что панель укоренённого ответа появляется после цитат
2. Убедиться, что отображаются следующие элементы:
   - **Сводка ответа** — ответ, основанный на доказательствах
   - **Бейдж статуса утверждения** — Эпистемический статус ответа
   - **Бейдж языкового режима** — Как выражен ответ
   - **Бейдж риска валидации** — Уровень риска с цветовой кодировкой
   - **Граница карта/территория** — Индикатор пограничного состояния
   - **Маркеры риска** — Список выявленных рисков
   - **Следующее действие** — Рекомендуемый следующий шаг
   - **Область** — Граница/контекст для ответа

**Ожидаемые результаты**:
- Все бейджи метаданных отображаются
- Ответ основан на процитированных доказательствах
- Маркеры риска чётко показаны

**Приоритет**: P0

### TC-004.7: Состояние отсутствующих доказательств

3. Убедиться, что показаны следующие предложения:
   - «Зарегистрировать больше документов»
   - «Попробовать другие поисковые запросы»
   - «Проверить содержимое документов»
4. Убедиться, что уверенность поиска показывает «отсутствует»

**Ожидаемые результаты**:
- Состояние отсутствующих доказательств чётко отображается
- Предложения практичны и полезны
- Пользователь понимает, почему результаты не были найдены
- Интерфейс не показывает пустую панель цитирования

**Приоритет**: P0

### TC-004.10: Высокая уверенность поиска
**Предусловие**: Документы содержат высоко релевантное содержимое для запроса

**Шаги**:
1. Зарегистрировать документ о «Политика удалённой работы: сотрудники могут работать из дома 3 дня в неделю»
2. Запрос: «Сколько дней сотрудники могут работать из дома?»
3. Убедиться, что уверенность поиска «высокая»
4. Убедиться, что цитаты показывают высокие оценки уверенности
5. Убедиться, что ответ сильно основан на доказательствах

**Ожидаемые результаты**:
- Уверенность «высокая» с зелёным бейджем
- Оценки цитирования высокие (>0,8)
- Ответ напрямую ссылается на доказательства
- Пограничное утверждение подтверждает сильную основу

**Приоритет**: P1

### TC-004.11: Низкая уверенность поиска
**Предусловие**: Документы содержат косвенно связанное содержимое

**Шаги**:
1. Зарегистрировать документ о «Общие офисные правила»
2. Запрос: «Какова конкретная политика дресс-кода?»
3. Убедиться, что уверенность поиска «низкая»
4. Убедиться, что цитаты показывают низкие оценки уверенности
5. Убедиться, что ответ включает соответствующие маркеры неопределённости

**Ожидаемые результаты**:
- Уверенность «низкая» с красным бейджем
- Оценки цитирования низкие (<0,5)
- Ответ выражает неопределённость
- Пограничное утверждение предупреждает об ограниченных доказательствах

**Приоритет**: P1

### TC-004.12: Содержимое пограничного утверждения
**Предусловие**: Запрос вернул результаты

**Шаги**:
1. Убедиться, что текст пограничного утверждения присутствует
2. Прочитать пограничное утверждение
3. Убедиться, что оно объясняет область доказательств и ограничения
4. Пример: «Этот ответ основан на 3 фрагментах документов из 2 документов. Доказательства охватывают общую политику, но могут не учитывать конкретные крайние случаи.»

**Ожидаемые результаты**:
- Пограничное утверждение читаемо для человека
- Объясняет, какие доказательства были использованы
- Отмечает ограничения или пробелы

**Приоритет**: P1

### TC-004.13: Маркеры риска в укоренённом ответе
**Предусловие**: Ответ содержит выявленные риски

1. Отправить запрос, который вызывает идентификацию риска

3. Убедиться, что каждый маркер риска — это читаемая метка
4. Примеры: `reductionism`, `map_territory_substitution`, `overgeneralization`

**Ожидаемые результаты**:
- Маркеры риска отображаются в виде списка
- Метки читаемы для человека
- Визуальное оформление отличается от других метаданных

## Конечные точки API

| Метод | Путь | Назначение |
|--------|------|---------|
| POST | `/projects/:projectId/retrieval/search` | Поиск документов по запросу |
| POST | `/projects/:projectId/retrieval/grounded-flow` | Полный укоренённый поиск (HTML) |

### Запрос поиска
```json
{
  "query": "What is the company policy on remote work?",
  "limit": 10
}
```

### Ответ поиска (200)
```json
{
  "hits": [
    {
      "snippet_id": "snip-1",
      "document_id": "doc-abc",
      "project_id": "proj-xyz",
      "score": 0.92,
      "confidence": "high",
      "citation_id": "cit-1",
      "matched_text": "Employees may work from home up to 3 days per week...",
      "source_label": "Remote Work Policy"
    }
  ],
  "retrieval_confidence": "high"
}
```

## Модели данных

### AvgRetrievalHit
```typescript
interface AvgRetrievalHit {
  snippet_id: string;
  document_id: string;
  project_id: string;
  score: number;                    // Relevance score (0-1)
  confidence: "low" | "medium" | "high";
  citation_id: string;
  matched_text: string;
  source_label: string;
}
```

### SearchDocumentsResult
```typescript
interface SearchDocumentsResult {
  hits: AvgRetrievalHit[];
  retrieval_confidence: "none" | "low" | "medium" | "high";
}
```

### AvgGroundedResponse
```typescript
interface AvgGroundedResponse {
  response: AvgStructuredResponse;
  grounding: AvgGroundedResponseBoundary;
}

interface AvgGroundedResponseBoundary {
  citations: AvgCitation[];
  grounded_claims: string[];
  interpretations: string[];
  unsupported_claims: string[];
  retrieval_confidence: "none" | "low" | "medium" | "high";
  boundary_statement: string;
}
```

## Обработка ошибок

| Сценарий | Ожидаемое поведение |
|----------|-------------------|
| Пустой запрос | Предотвратить отправку, показать ошибку валидации |
| Документы не зарегистрированы | Показать состояние «Отсутствующие доказательства» с предложениями |
| Запрос не вернул совпадений | Показать состояние «Отсутствующие доказательства» |
| Ошибка сети | Показать критическую ошибку, разрешить повтор |
| API возвращает 404 | Показать состояние «Отсутствующие доказательства» |
| API возвращает 500 | Показать критическую ошибку с деталями |
| Таймаут (>15 с) | Показать ошибку таймаута, разрешить повтор |
