## Сквозные аспекты

### Доступность (a11y)
Все пользовательские сценарии должны соответствовать стандартам **WCAG 2.1 AA**:
- Навигация с клавиатуры (Tab, Enter, Escape)
- Видимые индикаторы фокуса
- Совместимость с экранными читалками
- Коэффициенты контрастности цветов
- Текстовые метки сопровождают цветовые индикаторы

**Связанные E2E:** `accessibility.spec.ts`

---

### Визуальная регрессия
Все поверхности интерфейса отслеживаются на предмет визуальных изменений:
- Целевая страница
- Создание проекта
- Панель диалога
- Проверка утверждений
- Концептуальная карта
- Панель цитат
- Тёмная тема
- Мобильный viewport

**Связанные E2E:** `visual-regression.spec.ts`

---

### Производительность
Целевые показатели производительности для всех пользовательских сценариев:
- Загрузка страницы: < 3 секунд
- First Contentful Paint: < 1,5 секунд
- Time to Interactive: < 2,5 секунд
- Ответ API: < 500 мс
- Куча JavaScript: < 50 МБ
- Размер бандла: < 500 КБ

**Связанные E2E:** `performance.spec.ts`

## Стратегия тестирования

### Модульные тесты (Vitest)
Расположены в каталогах `tests/` пакетов:
- `avg-validation/tests/` — Валидация утверждений, классификация, оценка рисков
- `avg-graph/tests/` — Проекция графа, снимки, различия
- `avg-retrieval/tests/` — Репозиторий документов, поиск
- `avg-schemas/tests/` — Валидация контрактов схем
- `avg-agents/tests/` — Логика маршрутизации режимов

### E2E-тесты (Playwright)
Расположены в `e2e/tests/`:
- `project-creation.spec.ts` — UC-001
- `grounded-retrieval.spec.ts` — UC-004
- `claim-review.spec.ts` — UC-005
- `citation-panel.spec.ts` — UC-007
- `concept-mapping.spec.ts` — UC-006
- `accessibility.spec.ts` — Сквозные аспекты
- `visual-regression.spec.ts` — Сквозные аспекты
- `performance.spec.ts` — Сквозные аспекты

### Команды тестирования
```bash
pnpm test              # Запуск всех модульных тестов
pnpm test:e2e          # Запуск всех E2E-тестов
pnpm test:coverage     # Запуск тестов с покрытием
pnpm test:ai           # Запуск AI-эвалюаций
pnpm test:no-fairy-tale # Запуск проверок на галлюцинации
```

## Эпистемические принципы

Все пользовательские сценарии работают в соответствии со следующими основными принципами:

1. **Карта — не территория** — Модели являются проекциями, а не реальностью
2. **Никогда не представлять метафору как факт** — Утверждения «только метафора» должны быть чётко обозначены
3. **Основывать ответы на доказательствах** — Поиск предшествует генерации
4. **Валидация перед отображением** — Ошибки схем блокируют ответы
5. **Показывать неопределённость** — Гипотезы, риски и границы видимы
6. **Различать факт и гипотезу** — Статус утверждения всегда явно указан
