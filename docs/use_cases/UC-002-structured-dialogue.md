# UC-002: Структурированный диалог

## Обзор

| Поле | Значение |
|------|----------|
| **ID** | UC-002 |
| **Название** | Структурированный диалог |
| **Описание** | Пользователь вводит сырые мысли и получает структурированные ответы с эпистемическими метаданными (статус утверждения, языковой режим, риск валидации, граница карта/территория) |
| **Актор** | Пользователь с активным проектом |

| **Статус** | Заглушка (placeholder в WorkspaceShell) |

## Путь пользователя

### Точка входа
- Пользователь находится в активном рабочем пространстве проекта
- Пользователь нажимает вкладку "Dialogue" в навигации

### Основная концепция
AVG преобразует неструктурированный пользовательский ввод в **ответы, оформленные по контракту**, которые включают:
- **Статус утверждения** — Какой это тип знания?
- **Языковой режим** — Как это выражено лингвистически?
- **Риск валидации** — Насколько надёжно это утверждение?
- **Граница карта/территория** — Это модель или реальность?

### Поток действий
1. Пользователь выбирает вкладку "Dialogue"
2. Пользователь видит текстовое поле с плейсхолдером "Write the thought you want AVG to shape"
3. Пользователь вводит сырую мысль/идею
4. Пользователь отправляет мысль
5. Система обрабатывает ввод через конвейер AI-агентов
6. Система возвращает структурированный ответ с бейджами метаданных
7. Если ответ не проходит валидацию схемы, система показывает ошибку (никогда не показывает невалидный вывод)

---

## Режимы агентов

Система направляет пользовательский ввод в один из четырёх специализированных режимов:

| Режим | Ключевые слова | Поведение |
|-------|----------------|-----------|
| **creative** | creative, brainstorm, variants, alternatives, explore, invent, generate | Генерация вариантов и альтернативных направлений |
| **validator** | validate, claim, risk, metaphor, repair, boundary; или claim_status имеет metaphor_only/prohibited | Ответ с акцентом на валидацию, проверка утверждений |
| **architect** | architect, system, module, contract, milestone, pipeline, compose, route | Системный дизайн и компоновка ответа |
| **orchestrator** | По умолчанию, когда специалист не определён | Общая оркестровка |

## Тест-сценарии

### TC-002.1: Отображение вкладки диалога
**Предусловие**: Пользователь находится в активном рабочем пространстве проекта

**Шаги**:

1. Нажать вкладку "Dialogue" в навигации
2. Убедиться, что поверхность диалога рендерится
3. Убедиться, что текстовое поле присутствует
4. Убедиться, что кнопка отправки присутствует и активна

- Вкладка диалога выбрана и активна
- Текстовое поле рендерится с правильным плейсхолдером

**Приоритет**: P0

### TC-002.4: Валидация структуры ответа
**Предусловие**: Пользователь отправил валидную мысль

**Шаги**:

1. После получения ответа убедиться, что отображаются следующие элементы:
   - **Summary** — Основной текст ответа
   - **Claim Status Badge** — Один из: Definition, Working Distinction, Operational Marker, Indirect Sign, Hypothesis, Metaphor Only, Prohibited, Boundary Statement
   - **Language Mode Badge** — Один из: Direct, Operational, Conditional, Metaphor, Symbolic, Silence Required
   - **Validation Risk Badge** — LOW (зелёный), MEDIUM (жёлтый), HIGH (оранжевый), CRITICAL (красный)
   - **Map/Territory Boundary** — Preserved, Unclear или Violated
   - **Risk Markers** — Список идентифицированных рисков (если есть)
   - **Next Action** — Рекомендуемый следующий шаг
   - **Scope** — Граница/контекст для ответа

**Ожидаемый результат**:

- Все бейджи метаданных отображаются
- Цвета правильные для уровней риска валидации
- Граница карта/территория чётко указана
- Next Action — это действенный текст

**Приоритет**: P0

### TC-002.5: Отображение бейджа статуса утверждения
**Предусловие**: Ответ получен

**Шаги**:

1. Найти бейдж Claim Status
2. Убедиться, что отображаемый статус соответствует одному из 8 валидных значений:

| Статус | Значение | Визуальное оформление |
|--------|----------|----------------------|
| `definition` | Установленное определение | Нейтральный/синий |
| `working_distinction` | Рабочее различие | Нейтральный/зелёный |
| `operational_marker` | Операционный маркер | Нейтральный/бирюзовый |
| `indirect_sign` | Косвенный признак | Нейтральный/фиолетовый |
| `hypothesis` | Гипотеза (неопределённая) | Предупреждение/жёлтый |
| `metaphor_only` | Только метафора (не факт) | Предупреждение/оранжевый |
| `prohibited_positive_claim` | Запрещено | Критический/красный |
| `boundary_statement` | Пограничное утверждение | Информационный/серый |

**Ожидаемый результат**:

- Бейдж отображает человекочитаемую метку
- Цветовая кодировка соответствует эпистемическому статусу
- Утверждения "только метафора" и "запрещено" визуально отличаются от фактов

**Приоритет**: P0

### TC-002.6: Отображение бейджа языкового режима
**Предусловие**: Ответ получен

**Шаги**:

1. Найти бейдж Language Mode
2. Убедиться, что отображаемый режим соответствует одному из 6 валидных значений:

| Режим | Значение |
|-------|----------|
| `direct_description` | Прямое фактическое утверждение |
| `operational_description` | Описывает, как что-то работает |
| `conditional_description` | Описывает при определённых условиях |
| `metaphor` | Использует метафорический язык |
| `symbolic_pointer` | Указывает на что-то символическое |
| `silence_required` | Не может быть выражено языком |

**Ожидаемый результат**:

- Бейдж отображает человекочитаемую метку
- Режим чётко отличим от статуса утверждения

**Приоритет**: P1

### TC-002.7: Отображение риска валидации
**Предусловие**: Ответ получен

**Шаги**:

1. Найти бейдж Validation Risk
2. Убедиться в уровне риска и цвете:

| Уровень риска | Цвет | Значение |
|---------------|------|----------|
| `low` | Зелёный | Минимальные опасения |
| `medium` | Жёлтый | Некоторая неопределённость |
| `high` | Оранжевый | Значительный риск |
| `critical` | Красный | Не следует доверять |

**Ожидаемый результат**:

- Уровень риска соответствует метаданным ответа
- Цветовая кодировка правильная и доступная (контраст WCAG AA)

**Приоритет**: P0

### TC-002.8: Отображение границы карта/территория
**Предусловие**: Ответ получен

**Шаги**:

1. Найти индикатор Map/Territory Boundary
2. Убедиться в состоянии:

| Состояние | Значение | Визуальное оформление |
|-----------|----------|----------------------|

| `unclear` | Граница неоднозначна | Предупреждение/жёлтый |
| `violated` | Модель представлена как реальность | Критический/красный |

**Ожидаемый результат**:

- Состояние границы чётко отображается

**Приоритет**: P0

**Приоритет**: P0

### TC-002.10: Отображение маркеров риска
**Предусловие**: Ответ содержит маркеры риска

**Шаги**:

1. Отправить ввод, который запускает идентификацию риска
2. Убедиться, что маркеры риска отображаются в виде списка
3. Примеры маркеров риска:
   - `reductionism` — Упрощение сложных явлений
   - `map_territory_substitution` — Путаница модели с реальностью
   - `dogma` — Представление гипотезы как факта
   - `overgeneralization` — Широкие утверждения на основе ограниченных доказательств

**Ожидаемый результат**:

- Маркеры риска отображаются как читаемые элементы списка
- Каждый маркер имеет человекочитаемую метку
- Визуальный стиль отличает риски от других метаданных

**Приоритет**: P1

**Предусловие**: Пользователь находится на вкладке диалога

**Ожидаемый результат**:

- Каждый ввод запускает соответствующий специализированный режим

## API эндпоинты

| Метод | Путь | Назначение |
|-------|------|------------|
| POST | `/projects/:projectId/dialogue/page` | Рендер страницы диалога (HTML) |

### Тело запроса
```json
{
  "query": "Сырая мысль пользователя",
  "sessionId": "current-session-id",
  "messages": [{"id": "msg-1", "role": "user", "content": "Сырая мысль пользователя"}],
  "response": { /* AvgStructuredResponse */ }
}
```

### Ответ
- `content-type: text/html`
- Отрендеренная страница диалога со структурированным ответом

## Модели данных

### AvgStructuredResponse
```typescript
interface AvgStructuredResponse {
  id: string;
  project_id: string;
  session_id: string;
  message_id: string;
  summary: string;                        // Основной текст ответа
  scope: string;                          // Граница/контекст
  claim_status: ClaimStatus;              // Эпистемический статус
  language_mode: LanguageMode;            // Лингвистический режим
  validation_risk: DialogueValidationRisk; // Уровень риска
  risk_markers: string[];                 // Идентифицированные риски
  map_territory_boundary: MapTerritoryBoundaryState;
  next_action: string;                    // Рекомендуемый следующий шаг
  artifacts?: string[];                   // Ссылки на артефакты
}
```

### Перечисления
```typescript
type ClaimStatus =
  | "definition"
  | "working_distinction"
  | "operational_marker"
  | "indirect_sign"
  | "hypothesis"
  | "metaphor_only"
  | "prohibited_positive_claim"
  | "boundary_statement";

type LanguageMode =
  | "direct_description"
  | "operational_description"
  | "conditional_description"
  | "metaphor"
  | "symbolic_pointer"
  | "silence_required";

type DialogueValidationRisk = "low" | "medium" | "high" | "critical";

type MapTerritoryBoundaryState = "preserved" | "unclear" | "violated";
```

## Обработка ошибок

| Сценарий | Ожидаемое поведение |
|----------|---------------------|
| Пустой ввод | Предотвратить отправку, показать ошибку валидации |
| Ответ AI не проходит валидацию схемы | Показать критическую ошибку, никогда не показывать невалидный ответ |
| Ошибка сети | Показать сообщение об ошибке, разрешить повтор |
| Таймаут (>15с) | Показать ошибку таймаута, разрешить повтор |
| Невалидный JSON ответ | Показать ошибку парсинга, записать детали |

## Критерии приёмки

- [ ] Вкладка диалога рендерится с текстовым полем и кнопкой отправки
- [ ] Пустой ввод не может быть отправлен
- [ ] Структурированный ответ отображает все бейджи метаданных
- [ ] Бейдж Claim Status показывает правильный статус с правильным стилем
- [ ] Бейдж Language Mode показывает правильный режим
- [ ] Бейдж Validation Risk использует правильную цветовую кодировку
- [ ] Граница карта/территория чётко указана
- [ ] Ошибки валидации схемы показывают ошибку, а не невалидный ответ
- [ ] Маркеры риска отображаются как читаемый список
- [ ] Next Action отображается с контекстной подсказкой
- [ ] Маршрутизация режимов агентов работает на основе ключевых слов ввода
- [ ] Состояние загрузки показывается во время обработки
- [ ] Нет ошибок в консоли при нормальном потоке
