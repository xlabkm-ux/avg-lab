# UC-003: Регистрация документа

## Обзор

|Поле|Значение|
|-|-|
|**ID**|UC-003|
|**Название**|Регистрация документа|
|**Описание**|Пользователь регистрирует текстовое или markdown-содержимое как локальные для проекта документы-свидетельства для обоснованного поиска|
|**Актор**|Пользователь с активным проектом|
|**Приоритет**|P0 - Критический|
|**Статус**|Заглушка (заполнитель в WorkspaceShell)|
|**Связанный E2E**|Нет (поверхность — заполнитель)|

---

## Путь пользователя

### Точка входа

* Пользователь находится в активном рабочем пространстве проекта
* Пользователь нажимает вкладку "Документы" в навигации

### Основная концепция

Зарегистрированные здесь документы становятся **корпусом свидетельств** для Обоснованного поиска (UC-004). AVG сначала ищет в этих документах перед генерацией ответов, гарантируя, что ответы основаны на предоставленных пользователем свидетельствах со ссылками.

### Поток действий

1. Пользователь выбирает вкладку "Документы"
2. Пользователь видит форму с:

   * Поле ввода заголовка документа
   * Селектор типа источника (Простой текст, Markdown, Документ)
   * Текстовая область для содержимого
3. Пользователь заполняет форму и отправляет
4. Система регистрирует документ и генерирует фрагменты
5. Пользователь видит документ в списке с количеством фрагментов и оценкой токенов
6. Пользователь может нажать на документ для просмотра сгенерированных фрагментов и метаданных

**Предусловие**: Пользователь находится в активном рабочем пространстве проекта

**Шаги**:

### TC-003.2: Регистрация документа в формате простого текста

**Предусловие**: Пользователь находится на вкладке «Документы»

**Шаги**:

1. Введите заголовок: "Политика компании об удалённой работе"
2. Выберите тип источника: "Простой текст"
3. Вставьте или введите содержимое документа в текстовую область
4. Нажмите кнопку отправки
5. Наблюдайте за состоянием загрузки во время регистрации
6. Убедитесь, что документ появился в списке

**Ожидаемые результаты**:

* Документ успешно зарегистрирован
* Документ появляется в списке с:

  * Отображённый заголовок
  * Тип источника показан как "Простой текст"
  * Отображено количество фрагментов
  * Отображена оценка токенов
  * Временная метка регистрации
* Форма сбрасывается для следующей записи

**Приоритет**: P0

## Конечные точки API

|Метод|Путь|Назначение|
|-|-|-|
|GET|`/projects/:projectId/documents`|Список всех документов проекта|
|POST|`/projects/:projectId/documents`|Регистрация нового документа|
|GET|`/projects/:projectId/documents/:documentId`|Получить отдельный документ|
|GET|`/projects/:projectId/documents/:documentId/snippets`|Получить фрагменты документа|

### Запрос регистрации документа

```json
{
  "title": "Политика компании об удалённой работе",
  "source\_kind": "local\_text",
  "text": "Полное содержимое документа здесь...",
  "created\_at": "2026-05-22T18:00:00Z",
  "metadata": {
    "author": "Отдел кадров"
  }
}
```

### Ответ регистрации документа (201)

```json
{
  "document": {
    "id": "doc-abc123",
    "project\_id": "proj-xyz",
    "title": "Политика компании об удалённой работе",
    "source\_kind": "local\_text",
    "created\_at": "2026-05-22T18:00:00Z",
    "metadata": {}
  }
}
```

### Ответ списка документов (200)

```json
[
  {
    "id": "doc-abc123",
    "project\_id": "proj-xyz",
    "title": "Политика компании об удалённой работе",
    "source\_kind": "local\_text",

    "metadata": {}
```

## Модели данных

### AvgDocumentRef

```typescript
interface AvgDocumentRef {
  id: string;                    // Уникальный ID документа
  project\_id: string;            // ID родительского проекта
  title: string;                 // Заголовок документа
  source\_kind: DocumentSourceKind;
  created\_at: string;            // Временная метка ISO
  metadata?: Record<string, string>;
}

type DocumentSourceKind = "local\_text" | "local\_markdown" | "local\_document";
```

### StoredDocument

```typescript
interface StoredDocument {
  document: AvgDocumentRef;
  text: string;                  // Полное текстовое содержимое
  snippets: AvgSourceSnippet[];  // Сгенерированные фрагменты
}
```

### AvgSourceSnippet

```typescript
interface AvgSourceSnippet {
  id: string;                    // ID фрагмента
  document\_id: string;           // ID родительского документа
  project\_id: string;            // ID проекта
  ordinal: number;               // Позиция в документе
  text: string;                  // Текст фрагмента
  location?: string;             // Расположение внутри документа
  source\_label: string;          // Метка источника
}
```

## Обработка ошибок

|Сценарий|Ожидаемое поведение|
|-|-|
|Отсутствует заголовок|Предотвратить отправку, показать ошибку валидации|
|Отсутствует содержимое|Предотвратить отправку, показать ошибку валидации|
|Неверный ID проекта|Показать ошибку "Неверный формат ID проекта"|
|Ошибка сети|Показать сообщение об ошибке, разрешить повтор|
|Содержимое слишком большое (>1 МБ)|Показать ошибку ограничения размера|
|Дубликат документа|Разрешить (пользователь может зарегистрировать разные версии)|

## Критерии приёмки

* [ ] Вкладка «Документы» отображается с формой регистрации
* [ ] Заголовок и содержимое — обязательные поля с валидацией
* [ ] Селектор типа источника предлагает варианты: Простой текст, Markdown, Документ
* [ ] Регистрация документа создаёт документ в бэкенде
* [ ] Зарегистрированный документ появляется в списке с метаданными
* [ ] Отображаются количество фрагментов и оценка токенов
* [ ] Список документов показывает все зарегистрированные документы
* [ ] Представление деталей документа показывает полное содержимое и фрагменты
* [ ] Специальные символы и Unicode сохранены
* [ ] Большие документы обрабатываются без ошибок
* [ ] Нет ошибок в консоли при нормальном потоке
