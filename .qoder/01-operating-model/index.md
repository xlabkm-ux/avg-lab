# Operating Model Overview

Codex работает в трех режимах. Выбор режима определяет масштаб изменений и уровень изоляции.

## Режимы работы

| Режим | Когда использовать | Изоляция | Примеры |
|-------|-------------------|----------|---------|
| **Local** | Точечные исправления, отладка | Низкая | fix typo, run tests, debug |
| **Cloud** | Изолированные фичи, документация | Средняя | new component, docs update |
| **Subagent** | Параллельная работа | Высокая | independent features |

## Ключевой принцип

> **Не оптимизируйте количество агентов. Оптимизируйте границы.**

Малое количество хорошо изолированных агентов лучше, чем много агентов, редактирующих общий код.
